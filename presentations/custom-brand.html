How we keep the project's brain up to date — and the AI skills that power it

Remember your first day at a new job?
Don't know where the coffee machine is, how the work happens.

THE PROBLEM IS THE SAME FOR BOTH HUMANS AND AI AGENTS

We need to give them the context and training to be useful.
Without that guidance, the results are more or less random.

01

The challenge

CONTEXT IS SCATTERED EVERYWHERE

  • People's heads — decisions made in meetings, never written down
  • Slack threads — critical context buried 200 messages deep
  • Meeting notes — transcripts that nobody reads twice
  • Jira, Confluence, Miro — fragmented across tools
?
New joiners ramp slowly
↻
Decisions re-litigated
↓
Knowledge is lost over time

The solution

THE KNOWLEDGE BASE — A SINGLE SOURCE OF TRUTH

  • Git repository — Markdown files, version-controlled, PR-reviewed
  • Structured — specs, architecture, decisions, analysis, runbooks
  • CLAUDE.md — the briefing doc every AI session reads first
  • Not a wiki — it's code. Diffs, history, blame, branches all work
02

Why it matters

HIGH CONTEXT — THE ENABLER OF AN AI-NATIVE WORKFLOW

Knowledge is preserved
People leave. Meetings fade. Slack scrolls away. The KB captures decisions, context, and rationale — permanently, version-controlled, and searchable.
Humans & AI stay in sync
Both humans and AI agents read the same source of truth. No stale context, no repeated explanations. Everyone — person or machine — starts with full project knowledge.
Move at tomorrow's pace
The KB is the infrastructure that makes AI-assisted development work. Without it, agents guess. With it, they act with precision. Ensuring agents have the right context is the difference between AI as a toy, and AI as a force multiplier.
Without a knowledge base, you'll move at yesterday's pace. Or worse — you'll move fast at making a mess.

The workflow

EXPLORE, FORMALISE, EXECUTE, VERIFY

You don't pick a phase. You loop. Every feature, every sprint.

01
Explore
Vibe code it. Discover what you're building. AI accelerates prototyping.
02
Formalise
Stop. Extract specs. Lock decisions into git, not chat history.
03
Execute
AI implements within constraints. Specs, not vibes. KB provides the guardrails.
04
Verify
Automated gates. If spec ≠ code, it doesn't merge. Tests, drift checks, review.

Use case 1

GRANOLA CONSOLIDATION

Impact: Meeting outcomes are in the KB within minutes, not days. Every future AI session knows what was decided.

Use case 5

KEEPING DOCS AND CODE IN SYNC

Three skills that ensure documentation never silently falls behind.

/code-drift
Compares recent code changes against KB docs. Produces a gap report with specific action items.
/milestone-check
Checks the codebase against PRD acceptance criteria. Shows what's implemented and what's outstanding.
/spec-impact-analysis
Traces a specification change through to every affected code path. Prioritised action plan.
Impact: Drift is caught early and flagged with specific action items. No more "the docs say one thing, the code does another."

Use case 6

PROJECT MANAGEMENT VIA MCP

AI agents don't just read — they push to external services and record decisions as they go.

1 Ticket management — create, update, and triage Jira tickets without leaving the terminal
2 Risk & decision logging — decisions auto-recorded in the KB with timestamp, source, and rationale
3 Confluence publishing — PRDs, canvases, and changelogs pushed directly from markdown
4 Slack notifications — PR updates, stakeholder summaries posted to channels
MCP Servers Connected
Jira Create, search, update, transition issues
Confluence Create, update, publish pages
Slack Read channels, send messages, search
Granola Search meetings, read transcripts
Figma Read designs, extract context
Impact: No context switching. Decisions made in conversation are recorded in git and pushed to Jira/Confluence in the same action.

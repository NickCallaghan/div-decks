Architecture

Three layers, one file

STRUCTURE

Three Layers

Every deck is built from exactly three concerns, all embedded in a single HTML file.

  • CSS — theme variables, layout rules, reveal animations
  • HTML — semantic slide sections with typed classes
  • JS — the SlideEngine handles all navigation

THE HTML CONTRACT

deck.html 
<div class="deck">
  <section class="slide slide--title">
    <div class="reveal">...</div>
    <div class="reveal">...</div>
  </section>
  <section class="slide slide--content">
    <!-- next slide -->
  </section>
</div>

Reveal Animations

Staggered entrance with zero JavaScript

THE .reveal CLASS

style.css 
.slide .reveal {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.5s, transform 0.5s;
}
.slide.visible .reveal {
  opacity: 1;
  transform: none;
}
/* stagger via nth-child delays */
.slide.visible .reveal:nth-child(2) {
  transition-delay: 0.2s;
}

MECHANISM

Pure CSS Staggering

  • Wrap each element in a .reveal div
  • IntersectionObserver adds .visible to the slide
  • CSS nth-child rules stagger the delay
  • Elements fade up in sequence automatically
  • Respects prefers-reduced-motion

The SlideEngine

Navigation, scroll-spy, and chrome in ~80 lines

What SlideEngine Handles

Scroll SNAP NAVIGATION CSS scroll-snap + smooth scroll
Keys KEYBOARD INPUT Arrows, space, PgUp/Down, Home/End
Touch SWIPE SUPPORT Vertical swipe > 50px threshold
Chrome UI ELEMENTS Progress bar, dots, counter, hints

SLIDEENGINE INITIALIZATION

engine.js 
function SlideEngine() {
  this.deck = document.querySelector(".deck");
  this.slides = [].slice.call(
    document.querySelectorAll(".slide")
  );
  this.buildChrome(); // progress, dots, counter
  this.bindEvents(); // keyboard + touch
  this.observe();     // IntersectionObserver
  this.update();      // sync chrome state
}
new SlideEngine();

SCROLL SPY

IntersectionObserver

  • Watches each .slide element
  • 50% visibility threshold triggers .visible
  • Updates progress bar and dot indicator
  • No scroll event listeners needed

Navigation Chrome

Auto-generated UI that stays out of your way

TOP

Progress Bar

A 3px accent-colored bar at the top of the viewport. Width maps to current position as a percentage of total slides.

RIGHT

Dot Navigator

Clickable dots vertically centered on the right edge. Active dot scales up and takes the accent color. Frosted glass background.

BOTTOM BAR

Counter & Hints

  • Slide counter in bottom-right corner
  • Keyboard hints centered at bottom
  • Hints auto-fade after 4 seconds
  • All chrome is generated by SlideEngine
  • Nothing to add to your HTML manually
